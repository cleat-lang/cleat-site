Welcome to Cleat.

Cleat is a compiled, statically typed orchestration language for building auditable, safe AI agents. It compiles to native binaries through Go, ships with a full toolchain, and treats agents, guards, tools, streams, state machines, provenance chains, HTTP servers, and functions as first-class primitives—not library abstractions bolted on top of a general-purpose language.

What is Cleat?

Most backends that coordinate AI systems are written in Python or TypeScript, with orchestration libraries stitched together over the top. The result is a familiar one: the language does not know about your tools, your state transitions, or what your functions are actually allowed to touch. Bugs hide in glue code. Effects leak across boundaries. Auditability is an afterthought.

Cleat takes a different approach. The compiler itself understands what a tool is, what a stream is, what a state transition is, and what side effects your code is permitted to perform. Programs that violate these rules do not compile.

Tenet

If a guarantee matters at runtime, it should be enforced at compile time. Tests are good—types are better. The type system is the first line of defense against AI backends that do things they should not.

Design philosophy

Cleat is opinionated in a few specific ways:

  • Effects are declared, not discovered. Every function states what it touches with needs. The compiler refuses to let a pure function silently reach for the network.
  • Orchestration is syntax, not API. agent, guard, tool, stream, state, chain, server, and fn are keywords. Timeouts, retries, buffers, tool loops, and signing policies live in the language—not in decorators you can forget.
  • Auditability is built in. Provenance chains are cryptographically signed, append-only, and part of the core language. You do not glue an audit log on later.
  • Tests mock by substitution. using swaps a tool or effectful function for a typed replacement inside a test block. No mocking frameworks. No monkey-patching.
  • Native binaries, no runtime. Cleat compiles to Go and produces a single binary. No interpreter to ship, no VM to tune.

A taste

Here is a complete, type-checked Cleat program that defines a tool, runs it, and includes a test that mocks the tool entirely:

analyze.cleat 
import "std/io"
import "std/json"
import "std/http"

type Analysis = struct {
    files_changed: int,
    summary: string,
}

tool analyze(repo: string, pr: int)
    -> Result[Analysis, string]
    needs { net }
    timeout: 30s
    retry: 2
{
    let url = "https://api.example.com/${repo}/pulls/${pr}"
    let resp = http.get(url)?
    let data: Analysis = json.decode(resp.body)?
    return Ok(data)
}

test "analyze can be mocked" {
    using analyze = fn(repo, pr) {
        return Ok(Analysis { files_changed: 3, summary: "ok" })
    }
    assert(analyze("test/repo", 1).IsOk())
}

A few things to notice: the tool declares needs { net }, so the compiler knows it can hit the network. It has a timeout and a retry policy baked in. The ? operator propagates errors. And the test swaps the real analyze for a typed mock with a single using clause.

Install

Cleat is distributed as source. Build the compiler with Go 1.22 or newer:

bash 
$ git clone https://github.com/cleat-lang/cleat
$ cd cleat
$ go build -o cleat ./cmd/cleat
$ ./cleat --version
cleat 0.1.1-alpha

Drop the cleat binary somewhere on your PATH and you are ready to compile.

Your first program

Create a file called hello.cleat:

hello.cleat 
import "std/io"

fn main() -> int needs { io } {
    io.println("hello from cleat")
    return 0
}

Run it directly:

bash 
$ cleat run hello.cleat
hello from cleat

Or compile it to a native binary:

bash 
$ cleat build hello.cleat
$ ./hello
hello from cleat
Note

Notice that main declares needs { io }. If you remove that and try to call io.println, the compiler will stop you with a precise diagnostic pointing to the exact line.

Project status

Cleat is in alpha. The compiler, formatter, test runner, and standard library are implemented and tested. Real programs compile and run. The language is usable today for building auditable AI backends.

  • 826 tests across 11 internal packages
  • 15 standard library packages
  • 29 example programs
  • Built-in test runner, formatter, and Rustc-style diagnostics
  • Structured LLM output via compile-time JSON Schema extraction
  • Declarative HTTP servers with typed handlers
  • Agent supervisor with retry, timeout, and tracing
  • VS Code extension with syntax highlighting

Syntax and semantics may still change before 1.0. Breaking changes will be called out clearly in the changelog.
